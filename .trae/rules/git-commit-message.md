每条 commit message 遵循：`类型 简短的古典化描述 （可选）详细说明`
根据 git diff 内容自动选择诗意类型：

| 变更类型 | 诗意类型 | 意象 | 诗意描述 |
|:--------:|:--------:|:----:|:--------:|
| 新增功能 | 新章 | 开卷 | 如开新卷，添锦绣之章 |
| 修复错误 | 修误 | 补墨 | 如补墨迹，使字迹清晰 |
| 文档更新 | 文墨 | 添注 | 如添注释，增文采之韵 |
| 代码格式 | 雅致 | 调色 | 如调布局，求和谐之美 |
| 代码重构 | 重构 | 重装 | 如重装帧，使结构更佳 |
| 测试相关 | 校验 | 验色 | 如验墨色，保质量之优 |
| 构建工具 | 杂务 | 理房 | 如理文房，备笔墨纸砚 |

示例：
- 新增功能 → 新章: 素缃·朝霞主题初成
- 修复错误 → 修误: 补墨迹对比之误
- 文档更新 → 文墨: 添素缃意境之章
- 代码格式 → 雅致: 调素缃布局之韵
