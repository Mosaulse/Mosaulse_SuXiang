# 素缃项目 · AI 写作风格指南
## 项目简介
素缃是一对从东方古籍美学中生长出来的 VSCode 颜色主题，以「素绢、墨迹、朱砂、青花」四象配色体系构建。所有面向用户的文字——包括 README、Changelog、Git 提交信息、Issue 回复、文档更新——均需延续统一的古典文学文风。

## 核心文风定义
### 一、语言底色：文白相间
以现代中文为骨架，以古典文言为肌理。不是写古文，而是让白话文浸润在古文的气质中。
- **骨架**：现代中文，读者无需古文功底即可读懂
- **肌理**：四字短语、对仗句式、古典意象点缀其间
- **节奏**：长短交替，如呼吸般自然。短句如印章落款，掷地有声；长句如卷轴展开，舒缓从容

**示例对比：**

| 常规技术文档风格 | 素缃风格 |
|---|---|
| 修复了深色主题的配色问题 | 调整深色主题墨色层次，营造更深沉之古卷氛围 |
| 新增了语义高亮支持 | 新增 semanticTokenColors 完整配置，支持现代编程语言之语义高亮 |
| 优化了代码高亮效果 | 精进语法高亮，字符串、注释、变量等语义区分更加清晰 |
| Updated README | 缘起重书，素缃一卷焕然一新 |
| Fixed a bug | 墨韵微调，朱砂归位 |

### 二、意象体系：固定且自洽
所有比喻和意象必须从以下固定意象库中取用，不可随意引入不相干的比喻：

| 意象类别 | 可用词汇 | 用于描述 |
|---------|---------|---------|
| **纸帛类** | 素绢、宣纸、古卷、缣缃、帛书、尺牍、卷轴 | 背景、底色、整体视觉 |
| **墨迹类** | 墨香、墨色、墨痕、墨韵、褪淡、经年 | 前景文字、代码文本 |
| **朱砂类** | 朱砂、印红、钤印、落款、印章 | 关键字、强调元素、状态标记 |
| **青花类** | 青花、瓷纹、靛蓝、清冷、素雅 | 函数、类型、标题 |
| **书写动作** | 落笔、着墨、提按、分行布白、挥毫、展卷 | 代码编写、结构调整 |
| **文房雅物** | 文房四宝、笔墨纸砚、砚台、镇纸、水注 | 工具、依赖、配置 |
| **书卷场景** | 开卷、展卷、合卷、灯下、午后、烛影 | 安装、启用、关闭、使用场景 |
| **色彩动词** | 浸润、渲染、浸染、晕染、沉淀、褪淡 | 变化、优化、调整 |

### 三、句式模式
#### 版本更新条目的标准句式
每条更新遵循「动作 + 对象 + 古典收尾」的三段式：
动作 + 对象，古典意象收尾

**常用技术动作的古典化映射：**
| 技术动作 | 古典化表达 |
|---------|-----------|
| 新增 | 新增 / 添设 / 铺陈 |
| 优化 | 优化 / 精进 / 打磨 / 润色 |
| 修复 | 调整 / 修正 / 归位 / 归正 |
| 重构 | 重整 / 重构 / 重书 |
| 统一 | 统一 / 浑然一体 / 归一 |
| 删除 | 删削 / 去繁 / 削除 |

**古典收尾的常用句式：**
- ……更加清晰 / 醒目 / 分明 / 和谐
- ……层次井然 / 各归其位 / 无突兀之感
- ……如分行布白之章法
- ……温润如初雪宣纸
- ……如朱砂小印，钤于行侧
- ……如立范本而后临帖

#### 版本标题的标准格式
每个版本需要一个诗意的四字或六字副标题：

### vX.X.X — YYYY-MM-DD
**四字/六字诗意副标题**

**副标题范例：**
- 墨韵微调 · 语义高亮与活动栏
- 意境精进
- 意境升华
- 意境优化
- 主题微调与结构增强
- 朱砂归位 · 关键字高亮修正
- 素绢重织 · 底色焕新
- 青花入纹 · 类型高亮精进

#### 引言与过渡句
每个章节开头用一句古典引言作为引子，格式为：
> 古典引言。 ——出处

或化用典故（不必逐字引用，可改写）：
> 天地有大美而不言，四象有明法而不议。 ——化自《庄子 · 知北游》

**常用典故来源（按优先级）：**
1. 《诗经》——质朴之美
2. 《楚辞》——色彩与华美
3. 《说文解字》——字源与本义
4. 《论语》——实用与修身
5. 《礼记》——礼仪与规范
6. 《庄子》——自然与哲思
7. 《颜氏家训》——书卷与治学
8. 陶渊明诗文——闲适与雅趣

### 四、语气与态度
- **温润而非炫学**：引用典故是为了增添韵味，不是为了展示学问。引用后若需解释，用白话自然带出，不做学术考据
- **谦逊而非卑微**：称用户为「君」或「诸君」，自称用「此卷」「此主题」，不用「我」
- **从容而非散漫**：每句话都有信息量，不堆砌华丽辞藻。修饰是为技术服务的，不是为修饰而修饰
- **郑重而非死板**：像一位匠人介绍自己的作品——认真，但有温度

### 五、Git Commit Message 规范
每条 commit message 遵循以下格式：
> 类型 简短的古典化描述 （可选）详细说明，延续文白相间的风格

**类型标签：**
| 标签 | 含义 |
|------|------|
| 墨 | 代码逻辑、配置结构变更 |
| 色 | 配色调整、色值修改 |
| 笺 | 文档、README、注释更新 |
| 印 | 版本发布、标签 |
| 裁 | 依赖管理、构建工具 |
| 校 | 修复错误、纠正问题 |

**示例：**

色 深色主题朱砂色微调，印红更沉稳
暮色卷中关键字色值由 #D95B4A 调至 #C8524A，
更贴近古印泥之沉着质感。

笺 缘起重书，适配 v1.3 新增章节
版本纪事添 v1.3.1 条目，四象配色表更新语义高亮说明。

墨 重整 base.json，公共配色抽离复用
如立范本而后临帖，深浅两卷共用底色配置，
后续维护只需修一处而全局皆应。

校 归正 git.blame 批注前景色
溯源批注原与正文同色，难以辨识。
今以朱砂小印之色钤于行侧，一目了然。

### 六、绝对禁止
以下写法与素缃文风相悖，严格避免：
- 网络流行语（「YYDS」「绝绝子」「家人们」）
- Emoji 堆砌（一个章节最多出现一次 emoji，仅用于标题前缀）
- 技术文档的干涩语气（「Fixed bug #123」「Updated dependencies」）
- 过度口语化（「这个功能超好用」「搞定了一个大 bug」）
- 生硬堆砌典故（一段话中引用超过两处典故）
- 与项目意象无关的比喻（「像火箭一样快」「像钢铁侠的盔甲」）
- 在非标题位置使用感叹号
- 自称「我」或「我们」

### 七、适用场景速查
| 场景 | 风格要求 |
|------|---------|
| README 正文 | 完整古典文风，可引用典故，诗意收尾 |
| Changelog 条目 | 副标题 + 诗意句式，每条有古典收尾 |
| Git commit message | 类型标签 + 古典化描述，详述用文白相间 |
| Issue 回复 | 温和谦逊，称「君」，不过度引用典故 |
| PR 描述 | 说明变更意图，用古典意象收尾 |
| 代码注释 | 简洁白话，必要时点缀一两处意象，不过度 |

## 快速参考卡片
当你需要为素缃项目撰写任何文字时，先问自己三个问题：
1. **纸上有墨吗？** ——这段文字有没有实际信息量？不能只有修辞没有内容。
2. **墨中有骨吗？** ——技术事实是否准确清晰？古典修饰不能模糊技术表达。
3. **骨上有韵吗？** ——读起来是否有素缃特有的温润感？不能只有干巴巴的技术事实。

三者缺一不可。先有纸（载体），再着墨（内容），终成韵（风格）。
