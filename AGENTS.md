# 素缃项目 · AI 开发指南

## 项目概况

VSCode 古典配色主题扩展（`suxiang-theme`），以「素绢、墨迹、朱砂、青碧」四象配色体系构建。四个主题变体：素缃·光、素缃·夜、紫缃·光、紫缃·夜。

## 技术架构

### 主题文件结构

```
themes/
├── base.json           # 公共配色（semanticTokenColors + tokenColors）
├── Suxiang-Light.json  # 素缃·朝霞
├── Suxiang-Dark.json   # 素缃·暮色
├── Zixiang-Light.json  # 紫缃·晓烟
└── Zixiang-Dark.json   # 紫缃·烛影
```

- 所有主题文件通过 `"include": "./base.json"` 引用公共配置
- 每个主题文件**只包含与 base.json 不同的覆盖值**
- 修改公共配色编辑 `base.json`；修改单个主题编辑对应文件

### 开发命令

```bash
npm install           # 安装依赖
npx vsce package      # 打包为 .vsix
# VSCode 中按 F5 启动扩展开发宿主进行调试
```

### 主题清理脚本

```bash
python scripts/apply_include_and_clean.py
```

编辑主题后运行此脚本，自动移除与 base.json 重复的值，确保只保留差异。

## 写作风格

所有面向用户的文字（README、Changelog、Git 提交、Issue 回复）均需延续统一的古典文学文风。

### 语言底色：文白相间

以现代中文为骨架，以古典文言为肌理。不是写古文，而是让白话文浸润在古文的气质中。

### 意象体系：固定且自洽

| 意象类别 | 可用词汇 | 用于描述 |
|---------|---------|---------|
| **纸帛类** | 素绢、宣纸、古卷、缣缃、帛书、尺牍、卷轴 | 背景、底色、整体视觉 |
| **墨迹类** | 墨香、墨色、墨痕、墨韵、褪淡、经年 | 前景文字、代码文本 |
| **朱砂类** | 朱砂、印红、钤印、落款、印章 | 关键字、强调元素、状态标记 |
| **青碧类** | 青碧、翠色、温润、沉静 | 函数、类型、标题 |
| **书写动作** | 落笔、着墨、提按、分行布白、挥毫、展卷 | 代码编写、结构调整 |
| **书卷场景** | 开卷、展卷、合卷、灯下、午后、烛影 | 安装、启用、关闭、使用场景 |

### Git Commit Message 规范

每条 commit message 遵循：`类型 简短的古典化描述 （可选）详细说明`

**类型标签：**

| 标签 | 含义 |
|------|------|
| 墨 | 代码逻辑、配置结构变更 |
| 色 | 配色调整、色值修改 |
| 笺 | 文档、README、注释更新 |
| 印 | 版本发布、标签 |
| 裁 | 依赖管理、构建工具 |
| 校 | 修复错误、纠正问题 |

**示例：**

```
色 深色主题朱砂色微调，印红更沉稳
暮色卷中关键字色值由 #D95B4A 调至 #C8524A，
更贴近古印泥之沉着质感。
```

```
墨 重整 base.json，公共配色抽离复用
如立范本而后临帖，深浅两卷共用底色配置，
后续维护只需修一处而全局皆应。
```

### 版本标题格式

```
### vX.X.X — YYYY-MM-DD
**四字/六字诗意副标题**
```

副标题范例：墨韵微调 · 语义高亮与活动栏、意境精进、青碧入卷 · 四象焕新

### 绝对禁止

- 网络流行语（「YYDS」「绝绝子」「家人们」）
- Emoji 堆砌（一个章节最多一次，仅用于标题前缀）
- 技术文档的干涩语气（「Fixed bug #123」）
- 过度口语化（「搞定了一个大 bug」）
- 自称「我」或「我们」——用「此卷」「此主题」
- 在非标题位置使用感叹号

### 快速参考

撰写文字前自问三问：
1. **纸上有墨吗？** ——有没有实际信息量？
2. **墨中有骨吗？** ——技术事实是否准确清晰？
3. **骨上有韵吗？** ——读起来是否有素缃特有的温润感？
